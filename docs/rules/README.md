## 常见问题

### Q: 我应该先读哪篇文档？
A: 建议从 [项目概览](00-project-overview.md) 开始，然后根据你的需求选择阅读路径。

### Q: 文档中的代码示例可以直接使用吗？
A: 大部分示例可以直接使用，但可能需要根据你的环境稍作调整。

### Q: 如何判断自己是否理解了架构？
A: 尝试回答以下问题：
- Mini-SGLang 的主要组件有哪些？
- 请求的生命周期是怎样的？
- Radix Cache 是如何工作的？
- Overlap Scheduling 如何提升性能？

### Q: 编码规范必须严格遵守吗？
A: 是的，所有代码必须遵循编码规范，这是代码审查的重要部分。

### Q: 开发新功能时需要阅读所有文档吗？
A: 不需要，根据你的任务选择相关章节。建议至少阅读项目概览和相关架构部分。
